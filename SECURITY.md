# Security Policy

## Reporting a vulnerability

If you find a security issue, please do not open a public issue with exploit details.

Use one of these channels:
- Open a private GitHub security advisory for this repository.
- Or contact the maintainer directly through a private channel.

Include:
- A clear description of the issue
- Impact assessment
- Steps to reproduce
- Suggested remediation (if available)

## Scope

Security-sensitive areas include:
- API key handling and configuration
- External HTTP requests and data ingestion
- Prompt and tool invocation pathways
- Dependency and supply-chain integrity
